# CMOAI Engineering Loop 運用メモ (CMO AI Lite)

CMO AI Lite のUI/UX、エージェント連携、AIプロンプト、アウトプット体験を継続改善するための開発ループ。

## データフロー

```text
商品URL
  → 内部LP解析キャッシュ（本文・スクリーンショット・OCR・抽出ジョブ）
  → 事実DB
  → WHO-WHAT戦略
  → バナー画像テンプレ + 追加指示
  → copyBrief + promptJson
  → gpt-image-2
```

## 目的

- ユーザーが案件内DBを業務ツールに近い感覚で操作できる。
- 手元の Claude Code / Codex でリポジトリを開き、4スキルで作業を完遂できる。
- 5ステップの入出力・保存ルールが守られる。
- バナー制作では事実DBを読まず、選択WHO-WHAT・テンプレ・追加指示だけを使う。

## オーケストレーターの役割

1. 目的を1つに絞る。
2. 実装担当、レビュー担当、必要に応じてプロンプト/AI担当へ作業を分ける。
3. 変更対象ファイルを明確にする。
4. 実装後、レビュー観点に照らして確認する。
5. 日本語のコミットメッセージと本文で記録する。

## サブエージェントの役割

### UI/UX実装担当

- 対象: `src/ui/index.html`, `src/ui/app.js`, `src/ui/styles.css`
- 観点: DB表の読みやすさ、3工程(リサーチ/WHO-WHAT/バナー)の導線

### エージェント連携担当

- 対象: `docs/agent-operations.md`, `.claude/skills/`, `src/server.js` のAPI
- 観点: 4スキルで案件操作を完遂できること

### プロンプト/AI挙動担当

- 対象: `config/prompts/*.md`, `src/core/*ai.js`, `src/core/*store.js`
- 観点: 入力DB、出力DB、事実DBのバナー非読込、gpt-image-2固定

### レビュー担当

- 実装後に、ユーザー目線で使いづらい箇所を指摘する。
- 重大度は P0/P1/P2 で付ける。

## 5ループの初期方針

### Loop 1: 体制と記録

- 開発ループ、役割、レビュー観点を明文化。`AGENTS.md` から参照。

### Loop 2: エージェント連携

- 4スキルと `docs/agent-operations.md` の整合を守る。

### Loop 3: テーブル/DB操作

- 事実DB、WHO-WHAT、バナー案のテーブル操作を改善。

### Loop 4: アウトプット/制作導線

- バナー画像/プロンプトを見やすくする。

### Loop 5: プロンプト/AI挙動

- 事実抽出、WHO-WHAT、バナー生成、テンプレ化の入出力を確認。

## 日本語コミット規約

コミットメッセージは日本語で、1行目に目的が分かる短い要約を書く。

## 各ループの完了条件

- 構文チェックが通る。
- 変更点が目的に沿っている。
- 5ステップのデータフローが docs/skills/AGENTS.md で一致している。
