## 0. このガイドラインの目的

CMO AIの広告、LP、営業資料、SNS投稿、LINE投稿画像、ウェビナー資料、ホワイトペーパーを、誰が作っても同じブランドらしさで制作できる状態を作る。

CMO AIは「煽って売るダイレクトマーケ」ではなく、「当たる理由から設計する、ダイレクトマーケAIシステム」である。成果への勢いは出すが、根拠のない断言、情報商材感、過度な煽り、ヤンチャな印象は避ける。

このガイドラインで守るもの:

- 誠実で信頼感があること
- 勢いがあること
- 先進的であること
- 制作物全体に統一感があること
- ダイレクトマーケの強さを持ちながら、情報商材的に見えないこと
- 景表法リスクのある表示を制作段階で抑えること
- AIエージェントが制作してもブランド品質が崩れないこと

CMO AIは、ただ広告を制作するAIではない。商品事実、顧客理解、勝ちパターン、表現レギュレーションを整理し、当たる理由を設計してから制作する、ダイレクトマーケティングに特化したAIシステムである。

## 3. メッセージルール

### 3-1. 基本思想

CMO AIのコピーは「結果」だけを叫ばない。必ず「なぜ当たるのか」を添える。

基本構造:

1. 課題を明確にする
2. 当たらない理由を分解する
3. CMO AIが設計する要素を示す
4. 制作物に落とし込めることを示す
5. 行動につなげる

### 3-2. キャッチコピーの使い方

「そりゃ、当たるわ。」は、ブランドの強い一言として使う。ただし、成果保証のように見えないよう、必ず直後に理由や仕組みを置く。

OK例:

- そりゃ、当たるわ。商品事実、顧客心理、勝ちパターンから広告を設計するから。
- そりゃ、当たるわ。当たる理由から作る、ダイレクトマーケAIシステム。
- 勘ではなく、構造から作る。そりゃ、当たるわ。

NG例:

- そりゃ、絶対当たるわ。
- 誰でも必ず売れる。そりゃ、当たるわ。
- 放置でCVが爆増。そりゃ、当たるわ。

### 3-3. タグラインの使い方

タグライン「当たる理由から設計する、ダイレクトマーケAIシステム」は、ブランド説明が必要な場面で使う。

推奨配置:

- LPヒーローのサブコピー
- 営業資料の表紙
- ウェビナー資料の表紙/最終スライド
- ホワイトペーパーの表紙/裏表紙
- SNS固定画像やLINEリッチ画像の補足文

使い方:

- 初見の人向けには、キャッチコピーとタグラインをセットにする。
- 既存顧客向け資料では、タグラインよりも具体的な機能・導入効果を優先してよい。
- タグラインは勝手に言い換えない。短縮する場合は「当たる理由から設計する」までに留める。

## 4. ビジュアルルール

### 4-1. デザイン原則

1. 構造が見えること — ただ綺麗なだけでなく、リサーチ、戦略、制作、改善の流れが見えるデザインにする。
2. 勢いは出すが、煽らないこと — 強いコピー、シャープな構図、鮮やかなブルーで勢いを出す。赤黄の煽り、札束、炎、過度な成功者表現は避ける。
3. 余白で信頼感を作ること — 情報を詰め込みすぎない。BtoB SaaSとして、読む人が判断しやすい余白を取る。
4. 人間の判断とAIの処理能力を両方見せること — AIに丸投げではなく、マーケター、設計、レビュー、意思決定があることを表現する。
5. 媒体が変わっても同じブランドに見えること — ロゴ位置、色、見出しサイズ、CTA、図解スタイルを共通化する。

### 4-2. ロゴルール(※ロゴ未納品のためLite未適用)

ロゴはCMO AIの信頼性を担保する最重要アセット。以下を必ず守る。

基本ルール:

- ロゴは変形しない。比率を変えない。色を勝手に変えない。
- ロゴの上に文字や図形を重ねない。背景とのコントラストを確保する。
- ロゴ周辺には十分な余白を取る。小さすぎて読めないサイズで使わない。

推奨配置:

| 制作物 | ロゴ位置 |
| --- | --- |
| 広告バナー | 右下または左上。CTAと競合しない位置。 |
| LP | ヘッダー左上。ヒーロー内では必要に応じてタグラインと併記。 |
| 営業資料 | 表紙中央または左上。本文スライドではフッター右下に小さく配置。 |
| SNS画像 | 右下固定。小さすぎる場合はロゴマークのみ使用可。 |
| LINE投稿画像 | 視認性優先。ロゴより見出しを優先し、右下に配置。 |
| ウェビナー資料 | 表紙に大きく、本文はフッター。 |
| ホワイトペーパー | 表紙と裏表紙に配置。本文中は必要最小限。 |

禁止例:

- ロゴを斜めにする / 影やグローを強くかける / グラデーションで塗り替える / 背景パターンとして大量配置する / 文字間を変える / Lite・Pro・CAMPの表記を独自に組み替える

補足:

- ロゴマーク単体は、正式にアイコン用途として定義された場合のみ使う。通常はロゴタイプを含むロックアップを優先する。
- 正式ロゴのSVG/AIが納品されたら作例画像を差し替える。

### 4-4. タイポグラフィ

正式フォント未確定のため、以下を暫定ルールとする。

推奨:

- 日本語: Noto Sans JP / Source Han Sans / システムゴシック
- 英数字: Inter / Helvetica Neue / システムサンセリフ

ルール:

- 見出しは太く、短く。本文は読みやすさを優先し、過度に詰めない。
- 文字間を極端に広げない。ロゴの字間を真似して本文に適用しない。
- 明朝体、手書き風、ポップ体は原則使わない。
- 強調は太字またはブルーで行う。赤文字の多用は避ける。

### 4-5. レイアウト

推奨:

- 12カラムまたは明確なグリッドを使う。
- 見出し、説明、図解、CTAの順に視線が流れるようにする。
- 余白を広めに取り、BtoB SaaSらしい信頼感を出す。
- 図解は左から右、または上から下にプロセスが流れる構造にする。

避ける:

- 要素を斜めに散らす / 文字を詰め込みすぎる / CTAを複数置きすぎる / 無意味な装飾を増やす / 数字や成果だけを巨大に見せる

### 4-6. 写真・画像

使ってよい方向:

- 広告代理店やマーケティングチームの会議 / リサーチ、分析、設計、レビューの様子 / ダッシュボード、データ、フロー図 / 人間とAIが共同で作業している印象 / 清潔感のあるオフィス、PC、ホワイトボード

避ける方向:

- 札束、豪邸、高級車、時計 / 炎、爆発、過剰な矢印 / 煽り顔、驚き顔、過度な感情表現 / 怪しいセミナー感のある人物写真 / 黒金、赤黄、ギラギラした成功者イメージ / AIが魔法のように全自動で成果を出す表現

### 4-7. 図解ルール

CMO AIの図解は、必ず「当たる理由の設計」を伝える。

共通フロー:

1. 商品URL → 内部LP解析
2. 事実DB
3. WHO-WHAT戦略
4. バナー画像テンプレ + copyBrief + gpt-image-2
5. 表現レギュレーションチェック
6. 改善 / PDCA

図解の表現:

- 四角形、ライン、矢印、番号を中心にする。
- アイコンは線画・単色・ミニマルにする。
- 3Dや過度なグラデーションは使いすぎない。
- 各ステップに短いラベルを付ける。
- 1図解につき主張は1つに絞る。

### 4-8. イラストスタイル

標準スタイルは、プレミアム・エディトリアル・ジオメトリック。人間の判断とAIによる構造化・処理能力が組み合わさっていることを伝えるために使う。

このスタイルで表現すること:

- 広告代理店やマーケティングチームが、根拠を見ながら意思決定している状態
- 商品事実、顧客理解、戦略、クリエイティブ展開がつながる構造
- AIが魔法のように成果を出すのではなく、人間が判断し、AIが整理・展開・高速化している状態
- ダイレクトマーケらしいスピード感はあるが、情報商材的な煽りや過剰な成功表現ではない世界観

| 人物像 | 落ち着いたビジネス人物。 | 成功者風、インフルエンサー風、怪しいセミナー講師風。 |
| 姿勢 | 資料を見る、指し示す、議論する、設計する、レビューする。 | ガッツポーズ、叫ぶ、煽る、勝利を誇示するポーズ。 |
| 体型・形状 | 幾何学的で整理されたシルエット。丸みは残しつつシャープで洗練。 | マスコット化、子どもっぽい頭身、過度なデフォルメ。 |
| 服装 | BtoB SaaSに合う清潔感のある服装。 | 派手な柄、過度なカジュアル、金銭的成功を想起させる服装。 |

#### 4-8-2. モチーフルール

| モチーフ | 意味 | 使い方 |
| --- | --- | --- |
| 粒子波 | AIによる処理、データの流れ、展開の広がり。 | 背景や下部の流れとして薄く。可読性を邪魔しない濃度。 |
| ロゴ由来の斜め矢印・コーナー | 方向性、推進力、設計された流れ。 | 角のアクセント、区切り。実際のロゴを装飾化しない。 |
| ダッシュボード/カード | 分析、判断材料、施策管理。 | グラフ、リスト、指標を抽象化。実在しない数値や読める文字は入れない。 |
| ノード/経路線 | 顧客理解、セグメント、訴求、クリエイティブの接続。 | 点と線で情報が整理されていく様子。 |
| ファネル/プロセス | ダイレクトマーケティングの設計工程。 | 商品事実、顧客理解、戦略、制作、改善の流れを簡潔に。 |
| 小さなSignal Orange | 重要ポイント、注意、分岐、判断箇所。 | 点、ターゲット、警告、重要ノードとして少量。 |

#### 4-8-3. 配色ルール

背景はWhiteまたはCloud Grayを基本にし、暖色のクリーム、ベージュ、黄色寄りの背景は使わない。

推奨比率:

- White / Cloud Gray / Mist Gray: 55-70%
- CMO Navy / CMO Deep Navy: 15-25%
- CMO Blue: 10-18%
- CMO Green: 3-8%
- Signal Orange: 0-3%

使い方:

- CMO Blueは主役色。アクティブな線、人物の服、重要な図形、粒子波。
- CMO Navyは締め色。髪、ジャケット、主線、濃色ブロック。
- CMO Greenは補助色。レビュー、改善、承認、組織実装、CAMP文脈。
- Signal Orangeは注意や重要点の一点アクセントに限定。CTA色や大面積背景にはしない。
- 肌色は自然で控えめにし、全体の印象を暖色に寄せすぎない。

#### 4-8-4. 媒体別の使い方

| 媒体 | 推奨する使い方 |
| --- | --- |
| LP | ヒーローまたはセクション導入。人物、データパネル、粒子波、斜め矢印を組み合わせる。 |
| 営業資料 | 表紙、章扉、プロセス説明スライド。人物は控えめに、図解やダッシュボードを主役に。 |
| ホワイトペーパー | 表紙や章扉。余白を広く、粒子波は薄く背景化。 |
| SNS/LINE画像 | 人物を1〜2名に絞り情報量を減らす。小画面では粒子や細線を減らす。 |
| ウェビナー資料 | テーマ説明や導入スライド。チームで設計・レビューする印象を優先。 |

#### 4-8-5. イラストのNG表現

- AIが魔法のように広告成果を出している表現
- 札束、高級車、豪邸、派手な成功者、炎、爆発
- 赤・黄・金・黒を大面積で使った煽り系の配色
- 漫画/アニメ調、マスコット調、かわいすぎる人物
- 顔の表情で感情を強く煽る表現
- 実在しない数値やランキングを読める形で描くこと
- ロゴを分解して装飾パターンとして大量配置すること
- 粒子波や装飾を文字の背後に濃く置き、可読性を落とすこと
- Signal OrangeをCTA、見出し、大面積背景として使うこと

#### 4-8-6. AI生成時のプロンプト指針

イラストをAIで生成する場合は、以下の条件を必ず含める。

```
CMO AIのプレミアム・エディトリアル・ジオメトリックなイラストスタイル。
白/Cloud Gray背景、CMO Navy、CMO Blue、CMO Greenを中心に、Signal Orangeは小さな注意点アクセントのみ。
フェイスレスで落ち着いたビジネス人物。広告代理店やマーケティングチームが、データ、ダッシュボード、ファネル、ノード、粒子波を見ながら設計・判断している。
ロゴそのものは描かず、ロゴ由来の斜め矢印、コーナー、角度のあるモチーフを抽象的に使う。
文字、数字、ロゴ、成果保証、煽り表現、赤黄の派手な配色、札束、炎、漫画調、マスコット調は禁止。
```

## 5. 景表法・表示チェックルール(CMO AI自身の広告向け)

この章は制作時のリスク低減を目的とする。最終判断が必要な場合は、法務担当または専門家に確認する。

参考: 消費者庁「優良誤認とは」「有利誤認とは」「比較広告」「No.1表示に関する実態調査」「景品表示法関係ガイドライン等」

### 5-1. 基本原則

CMO AIの広告表現は、強くてもよい。ただし、根拠のない優位性、成果保証、条件を隠した有利訴求はしない。

必ず守ること:

- 成果保証に見える表現をしない。
- 実績数値は根拠資料とセットで管理する。
- No.1、国内初、業界初、最大級、最安などは事前レビュー必須。
- 比較広告は、比較条件、比較対象、調査時点、出典を明記する。
- 割引、無料、限定などの条件は見える位置に出す。
- 顧客の声は、実在性、掲載許諾、編集範囲を管理する。
- AI生成表現でも、公開前に人間が確認する。

### 5-2. リスク別チェック

| リスク | 表現例 | 対応 |
| --- | --- | --- |
| 高 | 必ず売れる、CVR改善保証、広告費が半分になる、No.1、業界初、最安 | 原則使用不可。使う場合は法務/責任者レビュー必須。 |
| 中 | 売れる広告を作る、成果につながる、改善する、効率化する | 文脈と根拠を確認。条件やプロセスを添える。 |
| 低 | 当たる理由から設計する、制作プロセスを構造化する、チームの制作体制を強くする | 基本使用可。ただし成果保証に見えないようにする。 |

### 5-3. No.1 / 推奨率 / 満足度表記

No.1表記は原則として事前レビュー必須。使用前に、調査主体・調査時期・調査対象者・比較対象・サンプル数・設問文・集計方法・表示文言と調査内容の一致・実利用者調査かイメージ調査か・注記の表示位置を確認する。

避けるべき例:

- 顧客満足度No.1 だけを大きく表示し、調査条件を示さない。
- 実利用者ではない調査を、実利用者の評価のように見せる。
- 比較対象が限定的なのに、業界全体No.1のように見せる。

### 5-4. 比較広告

比較広告を使う場合は、比較内容が客観的に実証されている・数値や事実を正確に引用している・比較方法が公正である・比較対象/条件/時点を示している・競合を不当に貶めない、を満たす。

OK例: 「当社調査、2026年7月時点、主要5サービスの公開機能比較に基づく。」
NG例: 「他社より圧倒的に優れている。」「この分野で一番使える。」「競合はもう古い。」

### 5-5. 根拠台帳

広告、LP、営業資料、SNS、ホワイトペーパーで使う実績・数値・比較・顧客コメントは、根拠台帳に紐づける。

台帳項目: 表示文言 / 表示媒体 / 根拠資料 / 対象期間 / 比較条件 / 注記文 / 承認者 / 有効期限 / 掲載URL・ファイル

## 6. AIエージェント用制作ルール

### 6-1. 基本プロンプト

CMO AI関連の制作物をAIエージェントに作らせる場合、以下を必ず渡す。

```
あなたはCMO AIのブランドガイドラインに従うクリエイティブ制作エージェントです。
CMO AIは「当たる理由から設計する、ダイレクトマーケAIシステム」です。

守るべきブランド人格:
- 誠実で信頼感がある
- 勢いがある
- 先進的である
- デザイン全体に統一感がある
- ダイレクトマーケの強さはあるが、情報商材的・ヤンチャ・煽りにはしない

- 結果だけを叫ばず、当たる理由や設計プロセスを示す
- 成果保証、過度な断言、根拠不明なNo.1表現を使わない

- 情報商材っぽくないか
- 成果保証に見えないか
```
