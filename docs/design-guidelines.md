# CMOAI デザインガイドライン(トンマナ)

CMOAI の Web UI 全体で使う色、文字、部品、レイアウトの基準。UI を変更するときは必ずこのファイルと `src/ui/styles.css` の design tokens を先に確認し、トークンの外で色やサイズを直書きしない。

## デザインの方向性

- 位置づけ: 日本のB2B SaaS(freee、SmartHR系業務ツール)に並ぶ、白基調・高情報密度のダッシュボードUI。
- **サイドナビ+ワークスペース**の構造で、常に「どの案件の・どの工程にいるか」が分かること。
- 主役はアウトプット(バナー画像)。クリエイティブは大きく見せ、管理系テーブルはコンパクトに保つ。

## ブランドカラー(ロゴ実測値)

ロゴ `src/ui/assets/logo.png` からの実測ドミナントカラー:

| 由来 | 実測値 | 用途 |
| --- | --- | --- |
| 「CMO」スレートグレー | `#606068` | ブランドインク。見出し・本文系ニュートラルの基準 |
| 「AI」ブルー | `#3890E0` | ブランドアクセント。主要アクション・選択状態・リンクの基準 |

### アクセントスケール(ブルー)

`--accent-*` トークンは logo blue `#3890E0` を 500 に据えたスケールとする。

| トークン | 値 | 用途 |
| --- | --- | --- |
| `--accent-50` | `#EEF6FE` | 選択行・アクティブナビの背景 |
| `--accent-100` | `#DCECFC` | ホバー背景、インフォバッジ背景 |
| `--accent-200` | `#B3D7F7` | 選択枠、タグ枠 |
| `--accent-400` | `#56A3E8` | グラフ・装飾、ホバー枠 |
| `--accent-500` | `#3890E0` | ブランド基準色。アイコン、フォーカスリング、選択インジケータ |
| `--accent-600` | `#2270BC` | **主要ボタン背景・テキストリンク**(白文字/白地でコントラスト4.5:1以上を確保) |
| `--accent-700` | `#1A578F` | 主要ボタンのホバー/押下 |

原則: `#3890E0`(500)は面と装飾に使い、**小さい文字に載せる青は600以深**を使う(アクセシビリティ)。

### ニュートラル(スレート)

`--slate-*` スケール(`styles.css:10-22`)を唯一のグレーとして使う。基準はロゴのスレート `#606068`(≒ slate-500〜600 の間)。黒 `#000` と純グレー(`#888` など)の直書きは禁止。

- 本文: `--text-primary`(slate-900) / 補足: `--text-secondary`(slate-500) / プレースホルダ: `--text-tertiary`(slate-400)
- 背景: キャンバス `--canvas`(slate-50)、カード `--surface`(white)、罫線 `--border`(slate-150)

### ステータスカラー

成功=緑 / 失敗=赤 / 進行・注意=黄 / 情報=青(`--success-*` ほか、`styles.css:34-46`)。ステータスは必ず「色+ラベル文字」で示し、色だけで意味を伝えない。

## タイポグラフィ

- フォント: `"Noto Sans JP", ui-sans-serif, ...`(既定スタック)。等幅は ui-monospace(プロンプト・JSON・パス表示)。
- サイズの段階: 12(補足・メタ) / 13(テーブル本文・フォーム) / 14(標準本文・ナビ) / 16(セクション見出し) / 20(ページタイトル) / 24(特大数値など)。この6段以外を増やさない。
- ウェイト: 400(本文)、500(ナビ・ラベル)、600(見出し・強調)。700以上は使わない。
- 日本語文言: です・ます調で簡潔に。ボタンは「動詞で終える」(例: 生成、追加、保存)。用語(商品マスター、事実、WHO-WHAT、広告テンプレ)を正とする。

## レイアウト

- 構造: 左サイドナビ(固定・折りたたみ可) + 上部パンくず/ページヘッダー + コンテンツ。
- **フラット原則(入れ子禁止)**: コンテンツ面は白ベタで、全体を包む「大きな白い枠(カード)」を作らない。サイドバー側を薄グレーにし、区切りは1px罫線のみ。カード扱いにするのは本当のカード(ダッシュボードカード、クリエイティブカード、ドロワー内ブロック、モーダル)だけ。テーブルは枠で囲わずコンテンツ面に直接置く。
- **ドロワー**: 画面右端にフラッシュした全高パネル(右側の角丸なし、左1px罫線+shadow-lg)。「枠の中のパネル」に見せない。
- サイドナビ: 幅 236px(折りたたみ時 64px)。セクションは「案件 / 作業 / 共有 / 設定」。作業は工程順(01 リサーチ → 02 WHO-WHAT → 03 バナー制作)に番号付きで並べ、現在地はアクセント色を使わず `--slate-150` の背景のみで示す(縦バーなし)。
- 余白: 4pxグリッド(4/8/12/16/24/32)。カード内パディング 20〜24px(窮屈にしない)。
- 角丸: `--r-sm`(8) 入力・バッジ / `--r-md`(10) ボタン・セル / `--r-lg`(14) カード・モーダル。
- 影: **面と影で区切る**。カードは罫線を最小限にし `--shadow-xs`(ごく薄い拡散影)で浮かせる。ホバーで `--shadow-sm` に持ち上げ、浮くもの(モーダル、ポップオーバー、ライトボックス)のみ `--shadow-lg`。
- キャンバス: `--canvas` はカードの白が浮いて見える明度(slate-50〜75)を保つ。カード同士の境界は影と余白で表現し、外周罫線に頼らない。
- トランジション: ホバー・フォーカスの状態変化は 120〜160ms ease。装飾アニメーションは付けない。

## コンポーネント原則

- **ボタン**: 主要=accent-600背景+白文字は**1画面1つ**(その画面の「次の一手」だけ)。副次=白背景+`--border-strong`枠、三次=テキストボタン(枠なし・ホバーで淡背景)。破壊的=赤系。AI実行系ボタンには実行中スピナーと二重実行防止を必ず付ける。
- **テーブル**: ヘッダーは背景を塗らず slate-500 の12px・500で軽く見せ、**スクロール時は上部に固定(sticky)**。行高はゆったり(44px目安)、行ホバーで slate-50 にハイライトし、**行内アクションボタンはホバー時のみ表示**(常時ボタンだらけにしない。削除等は誤操作防止も兼ねる)。セルはインライン編集(クリックで編集、Enter確定)。長文は `clip()`+詳細ペインで全文。
- **バッジ/ピル**: 淡色背景+同系の濃色文字(例: accent-50背景+accent-700文字)、角丸フル、12px・500。枠線は付けない。
- **フォーカス**: キーボード操作可能な要素は `--accent-500` の2pxリングを表示(outline)。
- **カード(クリエイティブ)**: バナー案・テンプレは**画像優先のカード**で見せる。サムネイルは幅いっぱい・アスペクト比維持・最小高180px。カード下部に タイトル/ステータスバッジ/操作 を1行ずつ。
- **詳細ペイン/モーダル**: 行・カードのクリックで右ペインまたはモーダル。生成画像は原寸プレビュー+クリックでライトボックス(拡大・前後送り)。
- **空状態**: 「何をすると何が入るか」を1文+主要アクションボタンで示す(例: 「商品を追加するとリサーチを開始できます」)。
- **通知/エラー**: 画面上部トースト。AI失敗時は原因文をそのまま出す(日本語エラーメッセージが実装済みのため)。

## アウトプット表示の原則(最重要)

1. 生成画像(`outputs/banners/**`)は一覧で「見て選べる」大きさで出す。テキスト行の添え物にしない。
2. バナー制作(03)は カードグリッド をデフォルト表示とし、テーブルは切り替え表示とする。
3. 詳細表示では 画像 → プロンプトJSON の順。JSONは折りたたみ既定。
4. 広告テンプレDBも同様に画像カード優先。参照画像・生成画像はライトボックスで原寸確認できること。

## 禁止事項

- トークン外の色・影・角丸の直書き
- ダークUIへの部分的切り替え(ターミナル領域のみ例外で黒背景可)
- 1画面に主要ボタンを3つ以上置くこと
- 画像をアイコンサイズ(80px未満)のサムネイルだけで完結させること
