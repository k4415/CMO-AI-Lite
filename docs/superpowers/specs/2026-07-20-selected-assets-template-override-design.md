# 選択素材をテンプレ構造より優先する例外ルール 設計

## 目的

バナー制作時にユーザーが選択したロゴ・商品画像・その他画像を、広告テンプレの画像スロット有無や役割に関係なく必ず画像生成へ渡す。一方、選択されていない素材と、テンプレにも選択素材にも存在しない視覚要素は追加しない。

## 決定事項

- テンプレ構造は原則として閉じた構造のまま維持する。
- ユーザー選択素材だけを、テンプレ構造制約より優先する唯一の例外とする。
- 選択された素材はすべて必須素材として扱う。
- 選択されていないロゴ・商品画像・その他画像は使用しない。
- 対応する既存スロットがあれば優先して使う。
- 対応スロットがない、または不足する場合は、選択素材に限って最小限の配置追加・置換を許可する。
- 選択素材を理由に、装飾画像、アイコン、背景モチーフ、ロゴ風文字など別要素を増やしてはならない。
- 画面表示や選択UIは変更しない。既存チェックボックスの選択結果を唯一の入力契約とする。

## データ契約

`promptJson.selectedAssetPolicy` に、選択素材の役割、件数、パス、必須性を決定論的に保存する。AIの返答に委ねず、保存済みバナー案の選択パスからサーバー側で組み立てる。

画像生成時には、実際に添付できた入力画像一覧を正として次を最優先で指示する。

1. 添付画像はすべてユーザー選択済みであり、完成画像へ反映必須。
2. `brand-logo` は正式ロゴ、`product` は商品実物、`reference` は選択参考素材として役割を維持する。
3. テンプレに対応スロットがなくても、選択素材だけは例外として最小限の位置へ配置する。
4. 添付されていないロゴ・商品・参考素材を生成しない。

テンプレにlogo image枠がない場合は、選択ロゴ用の決定論的な例外領域（1件目は右上 `top 3% / left 68% / width 29% / height 12%`）を画像プロンプトとOCRの両方で共有する。これにより、正式ロゴを任意位置へ置かせて確認不能にせず、同じ領域で正式ワードマークを検証し、不一致時の再生成を維持する。

## 失敗条件の変更

従来の `TEMPLATE_IMAGE_SLOT_REQUIRED`、`TEMPLATE_IMAGE_ROLE_MISMATCH`、`TEMPLATE_IMAGE_SLOT_CAPACITY_EXCEEDED` は、ユーザー選択素材に対しては制作失敗にしない。テンプレのテキストゾーンや基本トポロジーの逸脱は、従来どおり補正・検証対象とする。

## 受け入れ条件

- 画像スロット0件のテンプレでも、選択ロゴ・商品画像を含むプロンプトを生成できる。
- ロゴスロットのないテンプレでも、選択ロゴを必須素材として画像生成へ渡せる。
- スロット数を超える複数選択でも構造エラーにしない。
- 無選択時は選択素材例外を有効化せず、ロゴ・商品画像を追加しない。
- copyplan → prompt → image の通常経路と再生成経路で同じルールが使われる。
- ロゴ枠のないテンプレでも選択ロゴ用の例外領域をOCRし、正式ワードマークを検証できる。
- 既存の閉じたテンプレ構造、ロゴ同一性、OCR警告のテストを壊さない。
