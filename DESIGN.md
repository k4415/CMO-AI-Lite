## 1. Visual Theme & Atmosphere(視覚テーマと雰囲気)

**キーワード: 誠実 / 信頼感 / 勢い / 先進性 / 統一感**

- 「当たる理由から設計する、ダイレクトマーケAIシステム」。煽らず、構造で信頼させるBtoB SaaS。
- 白とCloud Grayの広い余白の上に、CMO Navyのテキストと、CMO Blueのアクセントが走る。
- 雰囲気の目標: 「業務に入れられる安心感」+「データと構造が見えるダッシュボード」。情報商材感(赤黄・金・煽り)は絶対に出さない。
- 装飾は「構造を伝えるもの」だけ許可: 粒子ドット(AIの処理・データの流れ)、斜めコーナー・矢印(推進力)、ノード/経路線(情報の接続)。いずれも薄く、可読性を邪魔しない。
- 密度: テーブル・一覧は情報密度高め(Notion/Airtable的)、ダッシュボード・サマリーは余白広め。

## 2. Color Palette & Roles(色彩体系と役割)

`styles.css` の CSSカスタムプロパティを唯一の定義とする。**新しい色を直書きで増やさない。**

### 2-1. ブランドカラー

2026-07-09、正式ロゴ(SVG)の実色に確定(それ以前の #04142C / #2460F0 は暫定値)。

| トークン | HEX | 役割 |
| --- | --- | --- |
| `--brand-navy` | #0C243A | 基調色(ロゴ文字色)。見出し、本文強調、アクティブナビの塗り、主線 |
| `--brand-deep-navy` | #071A2B | 締め色(派生)。限定使用(フッター・強調エリア) |
| `--brand-blue` / `--accent-500` | #2469FF | 主役アクセント(ロゴ主ブルー)。CTA、リンク、タブ下線、選択状態、グラフ主系列 |
| `--accent-600` | #0062F8 | ロゴ第2ブルー。ホバー・押下 |
| `--brand-orange` | #FF6B3D | Signal Orange。注意・重要ラベル・失敗表示**のみ**(0〜3%) |
| CMO Green | #088C84 | **Liteでは原則使わない**(機能的な成功表示 `--success-*` は例外として維持) |

### 2-2. ニュートラル(Navyトーン)

| トークン | HEX | 用途 |
| --- | --- | --- |
| `--slate-0` | #FFFFFF | カード面、基本背景 |
| `--slate-50` | #F7F8FA | Cloud Gray。セクション背景、thead |
| `--slate-150` | #E8E8EC | Mist Gray。境界線、非アクティブ面 |
| `--canvas` | #F0F2F6 | ページ背景(白カードが浮く濃さ) |
| `--slate-500` | #5F6C84 | 補助テキスト |
| `--slate-900` | #0C243A | 本文・見出し(=CMO Navy) |

### 2-3. アクセントスケール(CMO Blue系)

`--accent-50`(#EEF4FF 淡い選択面) → `--accent-100/200`(タグ地・ホバー面) → `--accent-500`(#2469FF 主役) → `--accent-600`(#0062F8 ロゴ第2ブルー・ホバー) → `--accent-700`(#0050CC 押下)。白文字は `--accent-500` 以上の濃さでのみ使用(コントラスト比 4.5:1 以上)。

### 2-4. ステータス色(機能色)

| 状態 | 使用トークン | 例 |
| --- | --- | --- |
| 実行中・情報 | `--info-*` / accent系 | 生成中、提案中 |
| 成功・完了 | `--success-*`(緑。機能色として最小限) | 画像完了、設定済み |
| 注意・保留 | `--warning-*`(Signal Orange系) | 未着手の警告、条件付き |
| 失敗・削除 | `--danger-*`(赤) | 生成失敗、削除操作 |

**面積比の目標(Lite)**: White・Cloud Gray 55–65% / Navy 20–25% / Blue 15–20% / Orange 0–3%。

### 2-5. データ可視化(グラフ・チャート)

- 主系列: `--accent-500`。第2系列以降: `--accent-200` → `--slate-300` → `--brand-navy` の順。
- 強調・例外・リスク点のみ Signal Orange の小さな点。
- 円グラフ・バーとも彩度の高い多色は使わない(青の濃淡+ニュートラルで構成)。

## 3. Typography Rules(タイポグラフィ)

### 3-1. フォントファミリ

```css
font-family: "Inter", "Noto Sans JP", ui-sans-serif, -apple-system, BlinkMacSystemFont,
             "Segoe UI", "Yu Gothic UI", Meiryo, sans-serif;
/* コード・ID・URL */
font-family: ui-monospace, SFMono-Regular, Consolas, monospace;
```

- 和文: Noto Sans JP(400/500/600/700をロード済み)。明朝体・手書き風・ポップ体は使わない。
- 欧文・数字: Inter優先(統計数値・金額・日付の視認性)。

### 3-2. サイズ・ウェイト階層

| 階層 | サイズ/ウェイト | 用途 |
| --- | --- | --- |
| Display | 24–28px / 700 | ダッシュボードの統計数値 |
| H1 | 20px / 700 | ページタイトル(1画面に1つ) |
| H2 | 15–16px / 700 | セクション・カード見出し |
| Body | 13px / 400–500 | 本文、テーブルセル |
| Meta | 11–12px / 500 | パンくず、補足、チップ、時刻 |
| Label | 11px / 600 | thead、サイドバーのセクションラベル |

### 3-3. 行間・字間

- 本文の行間: **1.6–1.8**(和文基準)。テーブルセルは1.5まで詰めてよい。
- 見出しの行間: 1.3–1.4。
- 字間: 基本 0(ロゴの広い字間を本文に真似しない)。英数字ラベル(thead等)のみ 0.02em まで可。

### 3-4. 禁則処理・改行

- `line-break: strict`、`overflow-wrap: break-word` を本文系に適用。
- URL・ID・パスは `word-break: break-all` + monospace。
- 長文セルは `-webkit-line-clamp` で2〜3行に丸め、全文は詳細ペインで見せる。
- テーブルの数値・日付列は `white-space: nowrap`。

### 3-5. 強調

強調は **太字** または CMO Blue。赤文字での強調は禁止(エラー表示のみ赤)。

## 4. Component Stylings(コンポーネント)

### 4-1. トップバー(グローバルヘッダー)※新設

- 高さ 56px、白面、下ボーダー `--border`。
- 左: 折りたたみトグル + 暫定ロゴ(タグラインは置かない。2026-07-09決定)。
- 右: OpenAI接続ステータス(ドット+ラベル)、ヘルプリンク、設定ショートカット。
- ※検索・通知・アカウントはモックアップ由来だが Lite には該当機能がないため**置かない**(確認用ページではプレースホルダとして描いてよい)。

### 4-2. サイドバー

- 幅 232px、白面、右ボーダー。セクションラベル(案件/作業/共有)は 11px / 600 / `--text-tertiary`。
- ナビ項目: 18px線画アイコン + 13pxラベル。アクティブ = `--brand-navy` 塗りつぶしピル(radius 10px)+白文字。ホバー(非アクティブ)= `--slate-100`。
- 最下部: ヘルプカード(「CMO AI Liteの使い方」)+接続ステータス+バージョン表記。
- 1100px未満で折りたたみ(アイコンのみ)。

### 4-3. ページヘッダー(タイトル行)

- パンくず(11px / tertiary)→ タイトル行: H1 + 右端にアクション(主ボタン1つまで+副ボタン)+ 統計チップ群(`#viewStats`)。
- サブ行: 案件名/商品名(12px / secondary)。更新時刻を出す場合はここの右端。

### 4-4. ボタン

| 種類 | 見た目 | 用途 |
| --- | --- | --- |
| Primary | `--accent-500` 塗り+白文字、radius 8px、hover `--accent-600` | 生成実行、保存(1画面1〜2個まで) |
| Secondary | 白面+`--border-strong`枠+Navy文字、hover `--slate-50` | 補助操作 |
| Ghost/Icon | 枠なし、hoverで淡い面 | 行内操作(削除・編集) |
| Danger | 赤は**確認ダイアログ内のみ**塗り。一覧内はアイコンのみ | 削除 |

- ラベルは動詞で簡潔に(「戦略生成」「画像生成」「保存」)。煽り文言禁止。

### 4-5. タグ・チップ

- **統計チップ**(`viewStatChip`): 白面+枠+radius full。`<b>数値</b>ラベル`。
- **カテゴリタグ**(事実5区分など): 淡色地+濃色文字、radius 6px、11px。色は青系濃淡で区別(特徴=accent-100地/accent-700字、メリット=accent-50/accent-600、実績=navy10%地/navy字、権威性=slate-150/slate-700、オファー=info系)。**5区分に5色相を使わない**(青×ニュートラルの濃淡のみ)。
- **ステータスチップ**: 提案中=accent淡、制作利用中=navy淡、完了=success淡、失敗=danger淡、実行中=info淡+スピナー。アーカイブ=グレー。

### 4-6. カード

- 白面、`--border` 1px、radius 14px、`--shadow-xs`。ホバーで持ち上がるのはクリック可能なカードのみ(translateY(-2px)+shadow-sm)。
- カード見出し行: H2 + 右端に補足リンク/メニュー。
- KPI/統計カード: Display数値+11pxラベル+(任意で)前回比や補足を12pxで。

### 4-7. テーブル

- カード内に格納(`.tableWrap` = カード面+radius+overflow hidden)。
- thead: `--slate-25` 地、11px/600/secondary、下線 `--border-strong`。
- 行: 高さ48px基準、hover `#F5F8FE`。選択行は `--accent-50`+左3px `--accent-500`。
- **横幅原則: 1280px以上では全列が横スクロールなしで収まる**。可変幅は本文系1列に集約し、他列は固定幅+truncate。収まらない場合のみ `.tableWrap` 内で横スクロール(ページ全体は絶対に横スクロールさせない)。
- 追加行(+ ◯◯を追加)はテーブル先頭行の控えめなゴーストボタン。

### 4-8. タブ

- 下線型フラットタブ: inactive=13px/secondary、active=700/`--text-primary`+下線2px `--accent-500`。背景・枠なし。

### 4-9. フォーム

- 入力: 白面+`--border-strong`枠、radius 8px、focus時 `--focus-ring`。ラベルは12px/600を上置き。
- モーダル: radius 14px、shadow-lg、タイトル16px/700、フッター右詰めボタン(副→主の順)。

### 4-10. 空状態・オンボーディング

- 空テーブル: 1行メッセージ(secondary)+必要なら誘導ボタン1つ。イラスト大面積は使わない。
- ガイドレール(はじめてのバナーまで): 完了=青チェック、現在=accent強調+CTA 1つ、未来=番号+グレー。

### 4-11. データサマリーパネルとビュー切替

各DBビュー（事実 / 戦略 / 制作 / 広告テンプレ）は、テーブルの上に**データサマリーパネル**（チャート帯）を持ち、表示を切り替えられる。

**ビュー切替(セグメントコントロール)**
- タブ行の右端に「サマリー / テーブルのみ」の2択セグメント(アイコン: チャート/テーブル)。
- サマリー = チャート帯(高さ160〜200pxのカード1〜3枚)+テーブル。**初期値**(第一印象で分析力を見せる)。
- テーブルのみ = チャート帯を非表示にし、作業に集中(Notion/Airtable的な機能性を損なわない)。
- 選択はビューごとに localStorage に保存し、次回も維持する。

**チャートは「次のアクションが分かるもの」だけ置く**(装飾グラフ禁止)。採用前に必ず3テストを通す:

1. **次のアクションを1つ指せるか**(見た人の行動が変わらないグラフは置かない)
2. **実数2つの文章で置き換えられないか**(置き換えられるなら文章+数字にする)
3. **件数が1〜5件でも成立するか**(Liteの実運用は少件数。件数バーは大抵ここで落ちる)

カタログ(2026-07-08改訂):

| ビュー | チャート | 指す次のアクション |
| --- | --- | --- |
| 事実 | **目安付きカバレッジゲージ**: 5区分それぞれに推奨最低件数(特徴3/メリット3/実績2/権威性1/オファー2)の目盛りを引き、「n/目安m」表示。達成区分は✓、最不足区分はOrange警告+「あと◯件」+「事実抽出を実行」CTA | どの区分をあと何件集めるか |
| 戦略 | **仮説検証マップ**: 戦略(非アーカイブ)ごとに「バナー案n・画像完了n」のバー。0件は「未検証」ラベル。上位5+残り件数表記 | 次にどの仮説でバナーを作るか |
| 制作 | 生成ファネル(案→プロンプト準備済→画像完了)+失敗件数(Orange/赤) + **「次に作るべきもの」カード**(未検証の戦略n件・テンプレ未選択のバナーn件+「バナー案を追加」CTA) | 詰まりの解消と次の制作対象 |
| ホーム | パイプライン3カードのミニリング(64px・中央に実数): リサーチ=5区分の目安達成リング(達成区分n/5)、戦略=仮説検証ドーナツ(検証済みn/全n)、制作=生成ステータスドーナツ(完了/生成中/失敗、中央=完了n) | 案件の詰まり(不足区分・未検証仮説・生成失敗)が円3つで一目 |

- 事実の推奨最低件数は app.js の定数(`FACT_CATEGORY_TARGETS`)で一元管理する(将来調整可)。
- ステータスドーナツ単体・件数のみの構成バー・種別内訳バーは3テストを通らないため**廃止**(ホームの小ドーナツのみ例外として維持)。

**チャートの描画ルール**
- 外部ライブラリ不使用。インラインSVG+CSSのみ(バー、ドーナツ、ファネル)。
- 配色は 2-5 のとおり青の濃淡+ニュートラル。警告・失敗点のみSignal Orange/赤。
- ドーナツは中央に合計値(Display数値)。バーには実数ラベルを添える(%だけにしない)。
- 初回描画時にバー幅/ドーナツをCSSトランジションで軽くアニメ(prefers-reduced-motion では無効)。
- データ0件のチャートは空の円やバーを見せず、1行メッセージ+CTAに置き換える。

**「無料でこれがもらえるのか」体験の原則**
- サマリーパネル・カウントアップ・丁寧な空状態・ガイドレールで、有償SaaS級の完成度を見せる。
- ただし誇張はしない: 見せるのは実データの構造と次のアクションであり、飾りの数字やダミーのスコアを出さない。

### 4-12. 装飾モチーフ(限定使用)

- 粒子ドット: `radial-gradient` 1pxドット、opacity 0.25以下、mask でフェード。ダッシュボード上部・ヒーロー背景のみ。
- 斜めコーナー/矢印コネクタ: パイプラインの流れ表示のみ。ロゴを装飾パターン化しない。

## 5. Layout Principles(レイアウト)

- 基本グリッド: 4pxベース。余白スケール: 8 / 12 / 16 / 20 / 24 / 32。
- 構成: サイドバー(232px固定)+メインエリア(max-width 1440px、左右24–32px)。
- ダッシュボードは12カラム相当のカードグリッド(2〜4カラム、gap 16–20px)。
- 視線の流れ: パンくず→タイトル→統計チップ→(タブ)→コンテンツ→CTA。図解・パイプラインは左→右。
- 1画面の主CTAは1つ。CTAを並べすぎない。

## 6. Depth & Elevation(深度と影)

- 影はNavyベース `rgba(4, 20, 44, α)` のみ。
- 階層: canvas(#F0F2F6) < カード(white + shadow-xs) < ホバーカード(shadow-sm) < ドロップダウン(shadow-md) < モーダル(shadow-lg)。
- 枠線と影を併用し「面で区切る」。区切り罫線を多用しない。グロー・強い発光は禁止。

## 7. Do's and Don'ts(ガードレール)

**Do:**
- 色は必ずCSS変数から使う。数値・構造・プロセスを見せる。余白で信頼感を作る。
- ステータス・件数など「今の状態」を常時可視化する(統計チップ、進捗)。
- 日本語UI文言は簡潔・実務的に。「〜できます」調で、根拠や条件を添える。

**Don't:**
- 赤・黄・金の大面積、グラデーション多用、札束/炎/成功者イメージ。
- Signal OrangeをCTA・見出し・大面積に使う。CMO Greenをブランド装飾に使う。
- 成果保証に見える文言(「必ず」「絶対」「爆売れ」)。No.1・最安等の根拠なし表示。
- 要素を斜めに散らす、文字詰め込み、意味のない装飾、ロゴの変形・パターン化。
- ページ全体の横スクロール。テーブルの右切れ。

## 8. Responsive Behavior(レスポンシブ)

- ブレークポイント: 1280px(テーブル全列表示の下限)/ 1100px(サイドバー折りたたみ)/ 900px(カードグリッド1〜2カラム化)。
- テーブルは1280px未満で `.tableWrap` 内横スクロール。カードグリッドは折り返し。
- ダッシュボードのコネクタ矢印は縦積み時に非表示。
- タッチ目標は最小40px。

## 9. Agent Prompt Guide(エージェント向け実装ガイド)

UIを実装・変更するAIエージェントは以下を守ること:

```
- 本書(DESIGN.md)と docs/creative-guideline.md を読んでから src/ui/ を変更する。
- 色は styles.css の CSSカスタムプロパティのみ使用。HEX直書き・新色追加は禁止
  (トークン自体の追加は DESIGN.md の更新とセットでのみ可)。
- サイドメニュー項目・機能・操作フローを変えない。見た目の変更に限る。
- 見出しは1画面1つ(workspaceHeaderのh1)。パネル内に同文言のh2を置かない。
- テーブルは1280px以上で右切れ・横スクロールなしに収める。
- Signal Orangeは注意・失敗表示のみ。緑はブランド装飾に使わない。
- 変更後は `node --check src/ui/app.js` と、必要に応じてブラウザで実表示確認を行う。
```

## 10. Lite 画面構成とデータフロー

CMO AI Lite の UI は次の画面のみを提供する（手動資料DB・広告文/台本/記事LP制作は UI から撤去済み。商品URLの内部LP解析は事実抽出の裏側で継続）。

| サイドバー | 画面 | 主なデータ |
| --- | --- | --- |
| ホーム | パイプライン・ガイドレール | 事実 / 戦略 / バナーの進捗 |
| 前提情報 | 商品・表現レギュレーション・画像 | products / expressionRules |
| リサーチ | 事実DB | facts（5区分） |
| 戦略 | WHO-WHAT 戦略 | strategies |
| 制作 | バナー案DB | banners |
| 広告テンプレ | バナーテンプレDB | adTemplates（creativeType=banner） |
| 設定 | OpenAI API キー | ローカル設定 |

**データフロー（典型）**

1. 商品登録（前提情報）→ 商品URL・画像を設定
